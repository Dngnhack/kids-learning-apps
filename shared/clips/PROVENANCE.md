# Bundled Audio Clips — Provenance

These `.mp3` voice clips are **generated by us** and **owned by Digital Legends**. They are the primary
voice path for the kids apps (played as static files via HTML5 `Audio`), replacing unreliable runtime
browser `speechSynthesis` on the primary path.

## How they were made
- **TTS engine:** Windows **SAPI** (`System.Speech.Synthesis.SpeechSynthesizer`), built into Windows 11.
- **Voice:** **Microsoft Zira Desktop** (Female, en-US) — a Windows *system* voice.
- **Encoder:** `ffmpeg` n7.1 (libmp3lame), the copy already bundled with the project's installed
  `@remotion/compositor-win32-x64-msvc` dependency. WAV → mono, 22050 Hz, 48 kbps `.mp3`.
- **Generator script:** `scripts/audio-clips/gen-clips.mjs` (deterministic, offline, $0). Re-runnable.
- **Settings:** SAPI rate −2 (slightly slower for clarity).

## Licensing (Harvey lens — CLEAN for public + monetization-adjacent apps)
- The voice used is a **Windows system voice**. We are not redistributing the voice model, font, or any
  third-party voice asset — we ship only the **audio output we generated**, which is **our content**.
- This is why **SAPI was chosen over Piper**: a downloaded Piper voice model would carry its own license
  (some are CC-BY-SA / NonCommercial, which would be unsafe for these apps). SAPI sidesteps that entirely.
- No CC-BY-SA / NonCommercial / "share-alike" obligations attach to these files.
- **No external/copyrighted audio** is included. Reward chimes remain WebAudio-synthesized (no files).

## What's here
- `clips-manifest.json` — engine/voice/format + every shipped clip (name, text, duration, bytes).
- Number building-block words: `zero` … `nineteen`, `twenty`…`ninety`, `hundred`, `thousand` (30).
- Letter NAMES: `letter-a` … `letter-z` (26). Accurate via TTS (letter names, not sounds).

## `cheers/` — ORIGINAL celebratory cheer clips (SHIPPED)
12 short, warm, **original/generic** kid-praise phrases, generated the SAME way (Windows SAPI / Zira →
ffmpeg → mono 22050 Hz 48 kbps mp3) and stored as `cheers/cheer-01.mp3` … `cheers/cheer-12.mp3` with a
`cheers/cheers-manifest.json`. Phrases: *Way to go! · Great job! · You did it! · Awesome! · Nice work! ·
Hooray! · Super! · You're a star! · Well done! · Keep it up! · Fantastic! · Wow!* (~1.4–2.2s each, ~134 KB total).
- **Use:** `playCheer()` in `apps/shared/clips.js` picks ONE at random per correct answer and plays it via
  the shared no-overlap clip player, **alongside** the synthesized WebAudio reward chime (which is unchanged).
- **CIPP / COPYRIGHT — CLEAN:** these are **our own words in our own SAPI voice**. **NO copyrighted or
  trademarked characters, names, jingles, brands, or copied clips** (no Bluey / Paw Patrol / Disney / etc.).
  Using third-party kids-IP audio would be copyright + trademark + false-endorsement infringement — none is
  present here. The generated output is owned by Digital Legends.

## `phoneme-candidates/` — NOT SHIPPED to the play path
26 letter-SOUND candidates + a manifest flagged `STATUS: CANDIDATE — NOT VERIFIED`. Build-time TTS
commonly **mangles isolated phonemes**. These are **not wired** into the letters app. An
**ear-check** is required before any candidate is promoted into this folder + enabled in the phoneme seam
(`apps/kids-letters/src/phoneme-audio.js` `CLIP_MANIFEST`). Until then, letter SOUNDS use the existing
interim TTS approximation / the record-real-clips plan.
